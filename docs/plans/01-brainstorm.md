# Brainstorm — 가정통신문 자동 작성기

> brainstorming skill 적용. 원격 무인 세션이라 자체 판단으로 확정.

## MUST — 꼭 필요한 기능 (5)
1. **7종 템플릿 선택 + 즉시 폼**: 카드형 템플릿 그리드에서 1종 선택 → 해당 템플릿 전용 필드(예: 견학 → 일시·장소·집결·해산·준비물·교통수단·인솔자·비상연락)가 폼으로 펼쳐진다.
2. **실시간 미리보기**: 폼 입력값이 우측 A4 미리보기에 디바운스 없이 즉시 반영. 미반영 칸은 회색 placeholder로 시각화.
3. **학교/학년/반/교사 머리글 localStorage 자동 채움**: 첫 입력 시 저장, 이후 모든 템플릿 상단에 자동 표시. "정보 초기화" 버튼으로만 삭제.
4. **PDF 다운로드 + 인쇄**: `html2pdf.js`로 A4 PDF 저장. 별도 "인쇄" 버튼은 `window.print()` 호출 — print CSS가 편집 UI를 숨기고 통신문만 출력.
5. **최근 통신문 5개 캐시**: 작성 직후 JSON으로 localStorage 저장, 사이드바에서 클릭 시 폼+미리보기 복원.

## SHOULD — 있으면 좋은 기능 (3)
1. **Gemini 본문 초안 1회 생성**: "본문이 막힐 때" 버튼 → 현재 폼 필드를 시드로 공용 프록시 호출. 결과는 본문 textarea를 교체. `thinkingBudget: 0` 으로 비용 최소.
2. **JSON 내보내기/가져오기**: 학교 머리글 + 최근 통신문을 JSON 1파일로 백업/복원. 학년 인수인계용.
3. **다크/라이트 토글**: 편집 UI만 토글 — 미리보기 통신문은 항상 흰 종이 유지.

## MUST NOT — 절대 금지 (3)
1. **학부모 전자 발송 어떤 형태도 금지** — 이메일 전송, SMS, Slack, 카카오 알림톡, mailto 링크 자동 채움 등 모두 차단. PDF 저장과 인쇄만 출력 경로.
2. **학생 개인정보 입력 칸 추가 금지** — 이름·생년월일·번호·연락처 칸 만들지 않는다. 가정통신문 본질상 학생 식별 정보는 불필요.
3. **Gemini API 키를 사용자에게 묻거나 코드/스토리지에 저장 금지** — 공용 프록시만 사용. 키 입력 UI 자체 없음.

## 결정 로그
- "다중 학년 한 번에 발송"은 SHOULD에도 넣지 않음 — scope creep.
- 본문 markdown 편집 vs plain textarea → plain textarea 채택. 인쇄 출력의 시각적 안정성 우선.
- 미리보기 = read-only 강제. inline 편집은 폼과 동기화 깨짐 위험.
